# Штукатурка PRO — MVP

Платформа для клиентов компании механизированной штукатурки: личный кабинет с прогрессом ремонта в реальном времени.

## Что реализовано в этом MVP

- ✅ Регистрация клиента (телефон + email + пароль) — `/register`
- ✅ Авторизация через NextAuth (Credentials) — `/login`
- ✅ Ролевая модель: `CLIENT`, `ADMIN`, `FOREMAN` (защита роутов через `middleware.ts`)
- ✅ Дашборд клиента: круговой прогресс проекта, метрики, карточки комнат — `/dashboard`
- ✅ Загрузка отчёта бригадиром (площадь, чек-лист, комментарий, фото) — `/dashboard/reports/new`
- ✅ API `/api/reports` — создание отчёта, автоматический пересчёт прогресса комнаты
- ✅ Хроника работ (timeline) с фото — `/dashboard/timeline`
- ✅ Полная схема БД (Prisma) под все фичи из ТЗ: материалы, документы, платежи, чат, чек-листы качества, AI-сводки, уведомления

## Интеграция: скан квартиры → чертёж → смета → платформа

Полная цепочка автоматизации онбординга:

```
Клиент сканирует комнаты (RoomScanner / любое LiDAR-приложение)
        ↓ экспорт
MeasureSquare (строит чертёж + считает смету по м²)
        ↓ webhook на /api/integrations/measuresquare/webhook
Штукатурка PRO — автоматически создаёт/обновляет:
  • Room (площадь каждой комнаты)
  • EstimateItem (построчная смета)
  • Document (PDF чертежа)
  • Project.area / roomsCount (пересчитано)
```

### Что нужно для запуска интеграции

1. Написать на `integration@measuresquare.com`, запросить группу и API-ключ (X-Application + secret).
2. Заполнить `MEASURESQUARE_APPLICATION_ID` и `MEASURESQUARE_SECRET_KEY` в `.env`.
3. В кабинете MeasureSquare Cloud указать **Notification URL**:
   `https://ваш-домен/api/integrations/measuresquare/webhook`
4. **Важно уточнить у MeasureSquare**: как передать наш `Project.id` как `external_reference` при создании проекта в их системе — это единственный способ связать их проект с нашим. Уточняется в Integration API Spec после получения доступа.
5. Файлы: `lib/measuresquare.ts` (HMAC-клиент), `app/api/integrations/measuresquare/webhook/route.ts` (приём данных).

### Альтернатива без MeasureSquare

Если решите не подключать MeasureSquare, а строить сканирование сразу в своём iOS-приложении на Apple RoomPlan API — эндпоинт `app/api/projects/[id]/import-scan/route.ts` уже принимает полигон стен и площадь напрямую с устройства, минуя посредника.

## Закрытая экономика объекта (для бригадира/админа)

Раздел `/foreman/economics/[projectId]` — полностью скрыт от клиента (проверка роли и в `middleware.ts`, и повторно на самой странице). Показывает:
- себестоимость материалов (использовано × закупочная цена компании, поле `Material.costPerUnit`)
- прочие расходы (`LaborCost` — зарплата бригады, транспорт, аренда техники)
- выручку (из сметы `EstimateItem` или фактических оплат `Payment`)
- автоматически посчитанную маржу и % рентабельности

Добавить расход: `POST /api/projects/[id]/labor-costs` с телом `{ description, amount }`.

⚠️ Важно: поле `Material.costPerUnit` (закупочная цена) нигде не отдаётся клиенту через клиентские API/страницы — только `pricePerUnit` (цена в смете). Если будете добавлять новые страницы с материалами, следите, чтобы `costPerUnit` не попадал в ответы для роли CLIENT.

## Прямая трансляция с объекта

Камера на площадке стримит по RTMP в **Mux** (облачный видеосервис) → в кабинете клиента встраивается HLS-плеер.

### Настройка

1. Зарегистрируйтесь на **mux.com**, создайте Access Token (Settings → Access Tokens) — получите `MUX_TOKEN_ID` и `MUX_TOKEN_SECRET`, впишите в `.env`
2. Бригадир/админ подключает камеру к проекту: `POST /api/projects/[id]/camera` — в ответ придёт `streamKey` и `rtmpUrl` **один раз**, их нужно сохранить
3. Эти данные вводятся в настройках самой камеры/энкодера (большинство IP-камер и экшн-камер с Wi-Fi поддерживают RTMP push; для телефона подойдёт приложение вроде Larix Broadcaster)
4. Как только камера начинает стримить, клиент автоматически видит трансляцию на `/dashboard/camera` — она подтягивается по `playbackId`, который безопасно публичен (в отличие от `streamKey`)

### Важно про безопасность
`streamKey` — секрет, который может использовать кто угодно для вещания от имени вашей камеры. Он возвращается только один раз в ответе API и не хранится в клиентском коде — храните его только в настройках самой камеры.

## Отзывы, заявки на просмотр и публичная информация

- **`/consultation`** — публичная форма записи на просмотр для новых (ещё не зарегистрированных) клиентов → создаёт `Lead`, виден менеджеру на `/admin/leads`
- **`/dashboard/review`** — клиент оставляет отзыв (`rating` + комментарий) → уходит на модерацию (`Review.published = false`)
- **`/api/reviews/[id]/publish`** (PATCH, только ADMIN) — публикация отзыва после проверки
- **`/otzyvy`** — публичная страница с уже опубликованными отзывами
- **`/technology`** — информационная страница о процессе механизированной штукатурки (этапы, преимущества) — чисто контентная, без базы данных

## Калькулятор материалов, комнаты, история смет

### Расширенный калькулятор (`lib/calculator.ts`)
- База смесей: Jatbay, Magma, Volma, Knauf MP 75 + возможность добавить свою (кнопка "Своя смесь" на странице комнаты)
- Расход у каждой смеси задан на 4 толщины слоя: 10/15/20/25 мм (`PlasterMix.consumption10mm...25mm`)
- Грунтовки: Бетоноконтакт, Knauf Tiefengrund, Mittelgrund — расход на м², авторасчёт канистр
- Армирующая сетка — по погонным метрам стыков (поле комнаты `meshJointsMeters`), авторасчёт рулонов
- Маяки — считаются автоматически по периметру стен + высоте потолка (шаг и длина профиля — константы в `CALC_CONSTANTS`, легко скорректировать)
- Откосы — отдельное поле `slopeLinearMeters`, считаются в погонных метрах, переводятся в кг смеси через среднюю ширину проёма

Базовые смеси/грунтовки досоздаются в БД автоматически при первом обращении к `/api/mixes` или `/api/primers` — отдельно сидировать не нужно.

### Комнаты (`/dashboard/rooms/[roomId]`)
Клик по карточке комнаты открывает подробную страницу:
- Своё название комнаты
- Площадь стен, площадь пола (справочно), площадь окон, периметр, высота потолков
- Погонные метры откосов и стыков под сетку
- Выбор смеси + толщины + грунтовки, кнопка "Рассчитать" — мгновенный расчёт материалов с ценами
- Загрузка 1–3 фото комнаты
- Последние отчёты бригады по этой комнате

Клиент видит комнату в режиме только чтения (без цен и калькулятора) — редактирование параметров и расчёт доступны только FOREMAN/ADMIN.

### Смета объекта (`/foreman/projects/[id]/estimate`)
- Кнопка "Пересчитать по комнатам" — собирает расчёты по всем комнатам проекта в единую смету (`EstimateItem`, source="CALCULATOR")
- Цена за единицу редактируется кликом прямо в таблице — правки сохраняются и не перезатираются при пересчёте
- "Сохранить в историю" — фиксирует снимок сметы с датой (`EstimateSnapshot`), историю можно открыть и посмотреть построчно

### Профиль и аватар (`/dashboard/profile`)
Загрузка аватара — `POST /api/users/avatar`, отображается кружком с инициалами, если фото не загружено.

⚠️ Как и другие фото/файлы в проекте, аватары и фото комнат сейчас используют `uploadPhotoStub` — заглушку, возвращающую placeholder-URL. Замените на реальную загрузку в Supabase Storage/Cloudinary перед продакшеном.

## Запуск локально

```bash
npm install
cp .env.example .env       # заполните DATABASE_URL и NEXTAUTH_SECRET
npx prisma migrate dev --name init
npx tsx prisma/seed.ts     # тестовые пользователи и проект
npm run dev
```
